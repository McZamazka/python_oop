## TypedCollection

Реализован Generic-класс коллекции:

```python
class TypedCollection(Generic[T])
```

Коллекция хранит объекты определенного типа.

Внутри используется:

```python
self._items: list[T]
```

Реализованные методы:

- `add()` — добавление объекта
- `remove()` — удаление объекта
- `get_all()` — получение списка объектов
- `find()` — поиск одного объекта по условию
- `filter()` — фильтрация объектов
- `map()` — преобразование объектов

Также реализованы:

- `__len__`
- `__iter__`
- `__getitem__`

Коллекция проверяет тип объектов при добавлении.

---

## Protocol

Реализованы два Protocol.

### Displayable

Требует наличие метода:

```python
display() -> str
```

---

### Scorable

Требует наличие метода:

```python
score() -> float
```

Классы недвижимости не наследуются от Protocol напрямую, но подходят под них благодаря одинаковым методам.

Это называется структурная типизация.

---

## TypeVar

Использованы:

```python
T = TypeVar("T")
R = TypeVar("R")
```

Также добавлены ограничения через `bound=`:

```python
D = TypeVar("D", bound=Displayable)
S = TypeVar("S", bound=Scorable)
```

Теперь TypedCollection может работать только с объектами, у которых есть нужные методы.

---

## Метод map()

Метод `map()` принимает функцию-преобразователь и возвращает новый список результатов.

Пример:

```python
titles: list[str]
prices: list[float]
```

Это показывает, что тип результата может изменяться.
